文档对象模型（Document Object Model）

DOM是W3C组织推荐的处理可扩展置标语言的标准编程接口（HTML和XML文档的编程接口）。
它在JS中的对象通常是window.document（注意，这个是各个浏览器所实现的）。

浏览器对象模型（Browser Object Model）

不同于DOM，BOM并非拥有官方标准，但随着互联网的发展，渐渐的在各大浏览器开发者中也形成了一系列的标准。通常被称作是BOM。
下面列举了一些常见的BOM对象。

  • - 作为一个浏览器中的JS运行时，window总是存在的，所有的JS全局变量与函数都是它的成员。（但要注意，在浏览器以外的环境中并非如此，例如Node.JS中为之对应的则是global）
  • - screen里保存着一些浏览器提供给当前页面运行时的一些用户屏幕、窗口相关的信息，但在可能的情况下也有可能会造成个人隐私泄露问题。
  • - location通常用来控制页面重定向，或查询当前协议、路由路径等功能。可以把它看做是浏览器正上方地址栏的在JS运行时中的映射。
  • - history通常用来后退或前进页面。可以把它看做是浏览器左上角页面控制按钮的在JS运行时中的映射。
    但要注意，由于用户历史记录里包含了大量的个人隐私信息，所以绝大多数浏览器都不会向JS运行时提供任何信息，例如Chrome就只会提供几个指定的方法。
  • - JS运行时可以通过navigator去查询浏览器的各项数据或者是状态，抑或是通过浏览器向用户申请摄像头、麦克风等一系列媒体流。
